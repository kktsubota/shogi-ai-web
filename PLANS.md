# PLANS.md - 将棋AI Web分析システム 開発ロードマップ

本ドキュメントは、クライアント完結型（サーバーレス）で動作するPC・スマホ対応の将棋AI分析Webアプリケーションの開発ロードマップです。
Phase 1 (PoC) から Phase 3 (機能拡張 & PWA) まで、順を追って実装を進めます。

---

## 🚀 開発フェーズ一覧

| フェーズ | 目的 | 主な成果物 |
| :--- | :--- | :--- |
| **Phase 1** | 最小動作検証 (PoC) | `tsshogi` + WASMやねうら王 + AobaNNUE のブラウザ連携 |

## 📌 Phase 1: 最小動作検証 (PoC)

### 目的
`tsshogi` で管理した局面データ（SFEN）を Web Worker 経由で `YaneuraOu.wasm` に渡し、`AobaNNUE` 評価関数を用いて単一局面の解析（評価値・最善手取得）ができる最小構成を構築する。

### タスクリスト
- [ ] **1.1 Web Worker / WASM 環境の構築**
  - [ ] WASM (SIMD有効) ビルドされた `YaneuraOu` を Web Worker 上でロードする構成の作成。
  - [ ] Web Worker とメインスレッド間の USI テキストコマンド（`isready`, `position`, `go` 等）ブリッジ層の実装。
- [ ] **1.2 評価関数 (AobaNNUE) のロードとキャッシュ**
  - [ ] `AobaNNUE` (v1.1) モデルファイルを Fetch し、WASM エンジンへ渡す処理の実装。
  - [ ] IndexedDB による評価関数ファイルのローカルキャッシュ機構の作成。
- [ ] **1.3 tsshogi と USI ブリッジの結合**
  - [ ] `tsshogi` をプロジェクトに導入し、局面の SFEN 生成およびルール判定（合法手等）を行う処理の実装。
  - [ ] `position sfen <SFEN>` → `go nodes 100000` 送出による思考結果（`info depth ... score cp ... pv ...`）のパース処理作成。
- [ ] **1.4 最小検証用 UI**
  - [ ] 基本盤面描画、解析開始/停止ボタン、評価値表示（数値）の画面構築。

### 完了条件
* ブラウザ上のボタン押下で将棋AIが思考を開始し、評価値および最善手（PV）がリアルタイムに画面更新されること。

## 📌 Phase 2: レスポンシブ UI & スマホ・パフォーマンス最適化

### 目的
PC（横画面）とスマホ（縦画面）で快適に操作できるレスポンシブ UI を構築し、iOS Safari 等のモバイル環境での発熱・メモリクラッシュ（WebProcessクラッシュ）対策を完了させる。

### タスクリスト
- [ ] **2.1 coi-serviceworker による COOP/COEP 対応**
  - [ ] GitHub Pages 上で `SharedArrayBuffer`（マルチスレッド動作）を有効化するため、`coi-serviceworker` を導入。
  - [ ] `crossOriginIsolated === true` の判定と、未対応環境での `Threads=1` シングルスレッド・フォールバック制御。
- [ ] **2.2 デバイス判定 ＆ パラメータ動的最適化**
  - [ ] 端末判定ロジックの実装（User-Agent / `navigator.deviceMemory` 等）。
  - [ ] **PC時:** `USI_Hash = 128MB〜256MB`, `Threads = CPUコア数 - 1`
  - [ ] **スマホ時:** `USI_Hash = 16MB〜32MB`, `Threads = 1〜2` に上限を自動適用。
- [ ] **2.3 レスポンシブ UI (React + Tailwind CSS) の構築**
  - [ ] **PCレイアウト (横画面):** 左「評価値グラフ」 / 中央「盤面」 / 右「解析データ・棋譜」の3カラム構造。
  - [ ] **スマホレイアウト (縦画面):** 上部「盤面」 / 下部「タブ切替（解析・グラフ・棋譜・設定）」の縦並び構造。
  - [ ] タッチ操作ミス防止（タップ範囲の拡大、誤スワイプ防止）のレイアウト調整。
- [ ] **2.4 バックグラウンド・省電力制御**
  - [ ] Page Visibility API によるタブ非表示・バックグラウンド移行時の自動 `stop` コマンド送出。
  - [ ] スマホ向け「指定ノード数（または指定深さ）到達時」の自動停止オプション機能。

### 完了条件
* iOS Safari / Android Chrome 上でメモリクラッシュせず安定動作すること。
* スマホ（縦持ち）と PC（横画面）の双方で最適化されたレイアウトで操作できること。

## 📌 Phase 3: 棋譜インポート・UX拡張 ＆ PWA

### 目的
将棋クエスト等の外部サービスやファイルからの棋譜インポート機能を拡充し、1局全体の自動分析機能および PWA（ホーム画面追加）対応を行う。

### タスクリスト
- [ ] **3.1 高機能 棋譜インポート UI / モーダル**
  - [ ] **ファイル選択 / ドラッグ＆ドロップ:** `.kif`, `.csa`, `.sfen` ファイルの読み込み対応。
  - [ ] **ワンタップ貼り付け (Clipboard API):** `navigator.clipboard.readText()` によるワンタップクリップボード読み込み。
  - [ ] **将棋クエスト（c-loft）連携:**
    - [ ] ユーザー名の入力・保存（`localStorage`）機能。
    - [ ] `https://c-loft.com/shogi/quest/` 経由での直近対局リスト（対戦相手・日時・手数）の自動取得および選択UI。
  - [ ] **tsshogi プレビュー:** 読み込み完了時に初手〜数手の指し手一覧プレビューを表示。
- [ ] **3.2 全局自動分析 ＆ 評価値グラフ**
  - [ ] 1局全体の各局面を連続で自動解析する「全局検討モード」の実装。
  - [ ] Chart.js / Recharts 等を用いた評価値推移グラフの動的描画。
  - [ ] グラフ上のポイント選択による該当局面への即座ジャンプ機能。
- [ ] **3.3 初期ロード UX ＆ PWA 対応**
  - [ ] 評価関数ダウンロード時のプログレスバー表示（`0%` 〜 `100%`）および軽量/標準モデル選択ダイアログ。
  - [ ] PWA 対応（`manifest.json` 設定、アイコン追加、オフライン動作）。
  - [ ] アセット更新検知（「新しいバージョンがあります」通知）とキャッシュ破棄ロジック。
- [ ] **3.4 CI/CD パイプライン (GitHub Actions)**
  - [ ] GitHub Pages への自動デプロイワークフロー構築。

### 完了条件
* 将棋クエストのユーザー名から対局を選んでワンタップでインポートし、1局全体の自動解析と評価値グラフ生成が行えること。
* PWA アプリとしてスマホのホーム画面から起動・運用できること。
